# v3.136.0 _2026-06-01_

## Flutter

- [#4271](https://github.com/Dart-Code/Dart-Code/issues/4271)/[#6064](https://github.com/Dart-Code/Dart-Code/issues/6064): The **Flutter: New Project** command now lets you choose which platforms to enable in the new project (with a "Don't ask again" option to skip for future executions).
- [#6048](https://github.com/Dart-Code/Dart-Code/issues/6048)/[#6055](https://github.com/Dart-Code/Dart-Code/issues/6055): Flutter device daemon timeouts have been adjusted to reduce the chance of warnings when the daemon starts up slowly but is not entirely unresponsive.
- [#6018](https://github.com/Dart-Code/Dart-Code/issues/6018): Due to very low usage and overlap with other features, the Flutter Outline view has been deprecated and will be removed in a future release. If you have concerns about this, please post a comment on [github.com/Dart-Code/Dart-Code#6019](https://github.com/Dart-Code/Dart-Code/issues/6019).

## Editor

- [#5968](https://github.com/Dart-Code/Dart-Code/issues/5968)/[#6062](https://github.com/Dart-Code/Dart-Code/issues/6062): The "Add '{packageName}' to dependencies" quick-fix no longer sometimes appears below the actions for ignoring the diagnostic.

## DevTools

- [#6051](https://github.com/Dart-Code/Dart-Code/issues/6051): Opening DevTools in the browser now logs more information about the URL mapping process to the extension log to help debug issues in remote workspaces.

## Misc

- [#6056](https://github.com/Dart-Code/Dart-Code/issues/6056): Many places that create timers in the extension now use `unref()` to ensure they do not delay extension shutdown.
- [#6009](https://github.com/Dart-Code/Dart-Code/issues/6009): Many automated tests have been merged or removed (where they test functionality that now lives - and is tested - in the SDK) to speed up CI runs.

## Features/Fixes Enabled with Future SDKs

The following items depend on changes to the Dart/Flutter SDKs that have not yet been released to stable channels. They may be available to test in the current beta/master channels and will show up in future SDK releases.

- [#6061](https://github.com/Dart-Code/Dart-Code/issues/6061): The "Remove constructor name" code action no longer incorrectly shows up on unnamed constructor invocations.
- [#4289](https://github.com/Dart-Code/Dart-Code/issues/4289): New code actions have been added to add/remove prefixes on import directives.
- [#6046](https://github.com/Dart-Code/Dart-Code/issues/6046): Many refactors now support "Refactor Anyway" if they would produce non-fatal errors, instead of being rejected.
- [#6052](https://github.com/Dart-Code/Dart-Code/issues/6052): Document Highlights (which show other instances of the symbol at the text cursor) now have the correct `kind` set so that reads/writes are coloured differently.
- [#6043](https://github.com/Dart-Code/Dart-Code/issues/6043): Go-to-Definition now works on explicit extensions.
